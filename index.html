Методика проверки гипотезы H₁

Объект и предмет исследования

Объект: взаимозависимые системы критической инфраструктуры (энергетика, водоснабжение, транспорт) на макроэкономическом уровне агрегации и процессы каскадного распространения отказов между ними под действием кратковременных шоков.

Предмет: методика количественной оценки каскадного риска на основе стохастического оператора \(K_q\) с динамической матрицей Леонтьева \(A(t)\), калиброванной по данным WIOD и SCADA, и протокола контролируемого сравнения с детерминированными операторами \(K_{\text{Leontief}}, K_{\text{cl}}, K_{DR}\) в маргинальной зоне нагрузки.

Гипотеза H₁

В маргинальной зоне нагрузки (не насыщённых сценариях кратковременных шоков) стохастический оператор \(K_q\) с СДУ‑динамикой и нелинейным порогом ёмкости обеспечивает прирост метрики полноты выявления каскадных отказов \(\Delta(s,m) \ge 0{,}25\) относительно каждого из детерминированных операторов \(K_{\text{Leontief}}, K_{\text{cl}}, K_{DR}\) не менее чем в 75 % сценариев при фиксированной матрице межотраслевых связей \(A\).

Шаг 1. Калибровка параметров модели

    > Матрица межотраслевых связей \(A\).
    Построить базовую матрицу Леонтьева по данным WIOD: \[ a_{ij} = \frac{X_{ij}}{\text{GO}_j}, \quad a_{ii} = 0. \] При необходимости нормировать \(A\) по спектральному радиусу до целевого значения \(\rho_\text{target} \le 0{,}95\). > Пороги пропускной способности \(C_j\).
    Оценить пороги перегрузки для секторов energy и water по SCADA‑данным (например, HAI ICS, Kelmarsh) как высокие квантили нагрузки (например, \(Q_{0.95}\) от нормированной мощности), для транспорта — по proxy‑данным DfT Road Safety / логистической статистике. > Волатильности \(\sigma_j\) для стохастического оператора \(K_q\).
    Для каждого сектора:
      >взять высокочастотные SCADA‑ряды \(x_j(t)\) в нормальном режиме; >оценить приращения \(\Delta \log x_j(t)\), снять автокорреляцию ARIMA‑фильтром; >оценить дисперсию шума HAC‑оценкой Newey–West и перевести её в безразмерную волатильность \(\sigma_j^\text{dim} = \sigma_j^\text{raw} / (1 - C_j)\).
    Для транспорта при отсутствии надёжных SCADA‑данных зафиксировать \(\sigma_\text{transport} = 0\) по протоколу no‑fabrication. > Фиксация pre‑registered параметров.
    Задать заранее: порог значимого прироста \(\delta\), порог узла \(\theta_\text{node}\), горизонт моделирования \(T\), шаг \(\Delta t\), параметр нелинейной деградации \(\alpha\), критерий маргинальности \(\eta_\text{marg} = 0{,}80\), уровень значимости \(\alpha_\text{stat}\).

Шаг 2. Конструирование сценариев и инициализация

    > Каталог сценариев.
    Сформировать конечное множество сценариев \(S\), включающее:
      >синтетические сценарии (например, полный/частичный отказ energy, перегрузка transport, деградация water); >исторические события (UCTE 2006, Texas 2021, India 2012, Baltimore 2024).
    > Задание начальных условий.
    Для каждого сценария \(s\) задать:
      >вектор начального состояния \(x_0(s)\) (нормированная загрузка секторов); >вектор шока \(u(s)\) и инициирующий сектор \(i_0\) на основе отчётов регуляторов и экспертной реконструкции; >горизонт моделирования \(T\) (например, \(T = 7\) шагов для маргинальной зоны).
    > Отбор маргинальных сценариев.
    Рассчитать для каждого сценария: \[ \text{margin\_ratio}(s) = \max_j \frac{x_{j,0}(s) + u_j(s)}{C_j}. \] В множество \(\mathcal{M}\) (маргинальная зона) включить только сценарии с \(\text{margin\_ratio}(s) < \eta_\text{marg} = 0{,}80\). Насыщенные сценарии использовать только для описательной валидации, без проверки \(H_1\).

Шаг 3. Численный эксперимент (экспериментальное воздействие)

На этом шаге проводится собственно эксперимент: калиброванная модель инфраструктуры подвергается искусственно задаваемым шокам \(u(s)\), и оценивается реакция системы под четырьмя операторами.

    > Единый вход/выход для операторов.
    Для каждого сценария \(s \in \mathcal{M}\) и каждого оператора \(m \in \{\text{Leontief}, \text{cl}, DR, q\}\) использовать единый формат входных данных: \[ (x_0(s), u(s), A, C, T, \delta, \theta_\text{node}). \] > Детерминированные операторы.
    Для \(K_{\text{Leontief}}, K_{\text{cl}}, K_{DR}\) выполнить по одному прогону (\(N_\text{runs} = 1\)) с расчётом траектории или финального состояния \(x_T^{(m)}(s)\) по их определению. > Стохастический оператор \(K_q\).
    Выполнить \(N_\text{runs}\) стохастических прогонов (например, \(N_\text{runs} = 1000\)) по схеме Эйлера–Маруямы: \[ x_{t+1} = \operatorname{clip}_{[0,1]}\bigl(x_t + u_t + A(t)x_t + \boldsymbol{\sigma}\odot\sqrt{\Delta t}\,\boldsymbol{\varepsilon}_t\bigr), \] \[ A(t)_{ij} = a_{ij}\,\exp\!\left(-\alpha \frac{\max(0,\, x_j(t) - C_j)}{1 - C_j}\right), \quad \boldsymbol{\varepsilon}_t \sim \mathcal{N}(0, I_n). \] Использовать детерминированную генерацию сидов (например, \(\text{seed}_r = \text{SHA256}(s, r)\)) для воспроизводимости результатов.

Шаг 4. Детекция каскада и расчёт метрик

    > Индикаторы каскада.
    Для каждого оператора \(m\), сценария \(s\) и прогона \(r\) определить индикатор каскада: \[ I^{(m)}(s, r) = \mathbf{1}\left\{\exists\, i \neq i_0 : \max_{t \le T} \bigl(x_i^{(m)}(t; s, r) - x_i^{(m)}(0; s, r)\bigr) \ge \delta \right\}. \] > Метрика полноты выявления каскадов.
    Для каждого оператора \(m\) и сценария \(s\) вычислить: \[ K^{(m)}(s) = \frac{1}{N_\text{runs}} \sum_{r=1}^{N_\text{runs}} I^{(m)}(s, r). \] Это оценка вероятности каскадного отказа на сценарии \(s\) по оператору \(m\). > Относительный прирост стохастического оператора.
    Для каждого baseline‑оператора \(m \in \{\text{Leontief}, \text{cl}, DR\}\) определить: \[ \Delta(s,m) = \frac{K^{(q)}(s) - K^{(m)}(s)}{K^{(m)}(s)}, \] при \(K^{(m)}(s) > 0\). Сценарии с \(K^{(m)}(s) = 0\) выделить отдельно (без вычисления \(\Delta\)).

Шаг 5. Статистическая проверка H₁ и вывод

    > Внутрисценарный статистический тест.
    Для каждого сценария \(s \in \mathcal{M}\) и baseline \(m\) использовать paired bootstrap по парам прогонов \((r_{K_q}, r_{K_m})\), чтобы получить бутстрэп‑доверительный интервал для \(\Delta(s,m)\) на уровне \(1 - \alpha_\text{adj}\) с поправкой Бонферрони (три сравнения: \(q\leftrightarrow \text{Leontief}, q\leftrightarrow \text{cl}, q\leftrightarrow DR\)). > Подтверждение H₁ на уровне сценария.
    Считать гипотезу \(H_1^{(s,m)}\) подтверждённой для пары \((s,m)\), если нижняя граница доверительного интервала для \(\Delta(s,m)\) не меньше 0.25: \[ \text{CI}_{\text{low}}(\Delta(s,m)) \ge 0{,}25. \] > Агрегация по сценариям (ConfirmationRate).
    Для каждого baseline \(m\) определить множество сценариев с определённым \(\Delta(s,m)\): \[ \mathcal{M}'_m = \{\,s \in \mathcal{M} : K^{(m)}(s) > 0 \,\}, \] и рассчитать долю сценариев, где \(H_1^{(s,m)}\) подтверждена: \[ \text{ConfirmationRate}^{(m)} = \frac{\#\{ s \in \mathcal{M}'_m : H_1^{(s,m)} \text{ подтверждена} \}}{|\mathcal{M}'_m|}. \] > Интерпретация результата для общей гипотезы H₁.
    Считать гипотезу \(H_1\) в целом эмпирически поддержанной, если для всех трёх baseline‑операторов выполняется \[ \text{ConfirmationRate}^{(m)} \ge 0{,}75, \quad m \in \{\text{Leontief}, \text{cl}, DR\}, \] при этом отдельно описать сценарии с \(K^{(m)}(s) = 0\) и насыщенные сценарии вне \(\mathcal{M}\) (как границы применимости методики).
