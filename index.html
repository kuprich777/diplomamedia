Фаза 0. Постановка задачи и фиксация критериев (до эксперимента)
0.1
Формулировка гипотезы
Количественный детектор каскадов (на основе непрерывного прироста $\Delta x_j$) чувствительнее классического (на основе пороговой бинаризации $x_j \geq C_j$).
Критерии зафиксированы до получения данных. Метрика, порог, условия — неизменны.
$$H_1: \frac{K_q - K_{cl}}{K_{cl}} \geq 0.25$$ $$H_0: \frac{K_q - K_{cl}}{K_{cl}} < 0.25$$
0.2
Целевая функция сравнения
$K$ — доля MC-прогонов, в которых метод зафиксировал каскадный эффект, затрагивающий хотя бы одну неинициирующую подсистему. ВЫХОД
$$K^{(m)} = \frac{1}{R}\sum_{r=1}^{R} I_m(s,r), \quad m \in \{cl, q\}$$ Классический: $I_{cl} = \mathbf{1}\!\left[\exists\, j \neq j_0,\; \exists\, t \leq T:\; x_j(t) \geq C_j\right]$ Количественный: $I_q = \mathbf{1}\!\left[\exists\, j \neq j_0:\; \max_{t \leq T}(x_j(t) - x_j(0)) \geq \delta\right]$
$C_j$ — порог перегрузки (q95), $\delta = 0.10$
0.3
Модель: стохастическое дифференциальное уравнение
Динамика сектора $j$ описывается СДУ с дрейфом (межотраслевое распространение) и диффузией (волатильность из SCADA). МЕТОД Euler–Maruyama
При $\sigma=0$, $\rho=0$, $\Delta t=1$, $T=1$ совпадает со старым дискретным оператором (тест: test_compatible_with_old_operator)
Непрерывная форма: $$dx_j = \underbrace{\left(\sum_i a_{ij} x_i - \rho_j x_j\right)}_{\text{дрейф}} dt + \underbrace{\sigma_j x_j\, dW_j}_{\text{диффузия}}$$ Дискретизация (Эйлер–Маруяма): $$x_j^{k+1} = \text{clip}_{[0,1]}\!\left(x_j^k + \left(\sum_i a_{ij} x_i^k - \rho_j x_j^k\right)\Delta t + \sigma_j x_j^k \sqrt{\Delta t}\, Z_j^k\right)$$
$Z_j^k \sim \mathcal{N}(0,1)$ i.i.d. | Реализация: sde_integrator.py

Фаза 1. Калибровка параметров модели (по реальным данным)
1.1
Калибровка волатильности $\sigma_j$
Из SCADA-данных нормального режима (без атак) вычисляется волатильность логарифмических приращений. ДАННЫЕ HAI ICS Dataset (1.3M строк, 1 сек/запись) ДАННЫЕ Kelmarsh Farm SCADA (36 файлов, 10 мин/запись)
Скрипт: scripts/calibrate_sigma.py → data/calibration/sigma_calibrated.json
$$\sigma_j = \frac{\text{std}(\Delta \log x_j)}{\sqrt{\Delta t}}$$ Результаты: $\sigma_{\text{energy}}^{\text{ICS}} = 6.54$ ч$^{-1/2}$ (HAI, P4_ST/HT)
$\sigma_{\text{energy}}^{\text{wind}} = 0.790$ ч$^{-1/2}$ (Kelmarsh)
$\sigma_{\text{water}} = 18.08$ ч$^{-1/2}$ (HAI, P3)
1.2
Калибровка порогов перегрузки $C_j$
95-й перцентиль нормированной нагрузки в нормальном режиме. Если $x_j \geq C_j$ — сектор «перегружен». ДАННЫЕ HAI ICS (energy, water) ДАННЫЕ DfT Road Safety (transport, 16K HGV incidents)
Скрипт: scripts/calibrate_capacity.py → data/calibration/capacity_thresholds.json
$$C_j = q_{0.95}\!\left(\frac{x_j}{x_j^{\text{nom}}}\right)$$ Результаты: $C_{\text{energy}} = 0.883$ (HAI, нормальный режим)
$C_{\text{water}} = 0.646$ (HAI, нормальный режим)
$C_{\text{transport}} = 0.928$ (DfT HGV, 2020–2024)
1.3
Калибровка матрицы зависимостей $A$
Коэффициенты Леонтьева из таблиц «затраты-выпуск» трёх стран. Опционально — байесовское обновление по прокси-рядам аварий. ДАННЫЕ WIOD 2016 NIOT (43 страны, 2014) МЕТОД Леонтьев + Bayesian conjugate normal
Скрипт: scripts/calibrate_A.py → data/calibration/A_leontief.json
Байесовский: matrix_doc/bayesian_calibrator.py → A_bayesian_posterior.json
$a_{\text{raw}}[i][j] = \dfrac{X[i][j]}{\text{GO}[j]}$ — Леонтьев $A = \frac{1}{3}(A_{\text{RUS}} + A_{\text{DEU}} + A_{\text{USA}})$, rescale max→0.5 Spearman $\rho = 0.962$ между странами Байесовское обновление: $$\mu_n = \frac{\sigma_\varepsilon^2 \mu_0 + n \sigma_0^2 \bar{a}_{\text{data}}}{\sigma_\varepsilon^2 + n \sigma_0^2}$$
Prior: WIOD | Likelihood: Texas+India proxy ($n=214$)
1.4
Динамическая матрица $A(t)$ — деградационный множитель
При перегрузке поставщика $j$ ($x_j \geq C_j$) его способность передавать нагрузку снижается — моделирует разрыв цепочки поставок. ФИКСИР. $\alpha$ (параметр скорости затухания)
Реализация: SDEIntegrator._compute_dynamic_A(). При $\alpha=0$: $A(t) = A^{\text{static}}$ (fast path).
$$\varphi_j(t) = \exp\!\left(-\alpha \cdot \frac{\max(0, x_j - C_j)}{1 - C_j}\right)$$ $$A(t)_{ij} = A^{\text{static}}_{ij} \cdot \varphi_j(t)$$
$\alpha=0$: статика | $\alpha=5$: $\varphi \approx 0.007$ при $x_j=1$
1.5
Проверка устойчивости: спектральный радиус
$\rho(A) < 1$ гарантирует сходимость (затухание каскада). Аналог $\rho(W) < 1$ в DebtRank. ВЫХОД
$\det(A - \lambda I) = 0 \;\Rightarrow\; \rho(A) < 0.95$ $\rho(A_{\text{bayes}}) = 0.194 < 1$ ✓
Каскад затухает за 3–4 шага

Фаза 2. Один прогон Monte Carlo (повторяется $R = 1000$ раз)
2.1
Генерация seed
Детерминированный seed из SHA256 — гарантирует воспроизводимость. ФИКСИР.
$$\text{seed}(s, r) = \text{int}\!\left(\text{SHA256}(s : r)_{[0:16]},\; 16\right)$$
2.2
Инициализация $x_0$ и шок $u$
Доменные сервисы возвращают базовое состояние. Шок $u$ применяется однократно на шаге $k=0$. УПРАВЛ. Сценарий $s \in S$ (определяет $j_0$, тип, амплитуду шока)
Пример: S3 transport +0.25 → $x_T^{\text{post}} = 0.583$; Baltimore transport +0.30
$x_0$ ← статус доменных сервисов Шок на шаге $k=0$:
$x^1 = \text{clip}_{[0,1]}\!\left(\text{drift} + \text{diffusion} + u\right)$
$u \in \mathbb{R}^N$ — вектор шока (только $k=0$)
2.3
Интегрирование СДУ: $T$ шагов Эйлера–Маруямы
На каждом шаге: дрейф (межотраслевое распространение через $A$) + диффузия (калиброванная стохастика $\sigma_j$). Горизонт: $T=7$ (маргинальные) или $T=50$ (насыщенные). ФИКСИР. $A$, $\sigma_j$, $\rho_j$, $\Delta t$
Реализация: SDEIntegrator.run() — sde_integrator.py
Для $k = 0, 1, \ldots, T-1$: $$x_j^{k+1} = \text{clip}\!\left(x_j^k + \underbrace{\left(\sum_i a_{ij} x_i^k - \rho_j x_j^k\right)\Delta t}_{\text{дрейф}} + \underbrace{\sigma_j x_j^k \sqrt{\Delta t}\, Z_j^k}_{\text{диффузия}}\right)$$ $Z_j^k \sim \mathcal{N}(0,1)$ — i.i.d. из PRNG(seed)
2.4
Детекция каскада
По траектории $\{x_j^k\}_{k=0}^T$ параллельно вычисляются два индикатора. ВЫХОД $I_{cl}$, $I_q$
Реализация: SDEIntegrator.detect_cascade()
Классический (пороговый): $I_{cl} = \mathbf{1}\!\left[\exists\, j \neq j_0,\; \exists\, k:\; x_j^k \geq C_j\right]$
$C_E=0.883$, $C_W=0.646$, $C_T=0.928$ Количественный (непрерывный): $I_q = \mathbf{1}\!\left[\exists\, j \neq j_0:\; \max_k(x_j^k - x_j^0) \geq \delta\right]$
$\delta = 0.10$
2.5
Запись результатов прогона
Траектория, индикаторы, $\Delta R$ — всё сохраняется в JSON. ВЫХОД
Прогон $r$: $(I_{cl,r},\; I_{q,r},\; \{x_j^k\}_{k=0}^T,\; \Delta R_r)$ $\Delta R_r = \sum_j w_j(x_j^T - x_j^0)$
→ results/mc_runs/
▼ повторить $R = 1000$ раз
Фаза 3. Агрегация метрик
3.1
Полнота выявления каскадов $K$
ВЫХОД $K_{cl}$, $K_q$
$K_{cl} = \frac{1}{R}\sum_{r=1}^R I_{cl,r}$   $K_q = \frac{1}{R}\sum_{r=1}^R I_{q,r}$
3.2
Статистики прироста риска
ВЫХОД mean, p95, распределение max_delta
$\overline{\Delta R}$, $p_{95}(\Delta R)$, $\text{max\_delta}_r = \max_{j \neq j_0}(x_j^T - x_j^0)$

Фаза 4. Проверка гипотезы
4.1
Вычисление критерия и решение
$\Delta\% = \frac{K_q - K_{cl}}{K_{cl}} \times 100\%$ $\Delta\% \geq 25\%$ → $H_0$ ОТКЛОНЯЕТСЯ ✓ $\Delta\% < 25\%$ → $H_0$ не отклоняется
4.2
Результаты: подстановка
$R=1000$, $T=7$, $\alpha=0$, Эйлер–Маруяма
S3: $\frac{0.883-0.595}{0.595}=+48.4\%$ ≥ 25% ✓ S4: $\frac{1.000-0.620}{0.620}=+61.3\%$ ≥ 25% ✓ Baltimore: $\frac{0.884-0.606}{0.606}=+45.9\%$ ≥ 25% ✓ Europe 2006: $\frac{1.000-0.038}{0.038}=+2532\%$ ✓ India 2012: $\frac{1.000-0.142}{0.142}=+604\%$ ✓ S5 combined: $\frac{1.000-0.832}{0.832}=+20.2\%$ < 25%

Фаза 6. Валидация на реальных авариях
6.1
Метод: модель получает только силу удара
Входные параметры $(x_0, u)$ — из регуляторных отчётов. Модель не знает, какие секторы деградировали. Предсказание сравнивается с документированной реальностью. ДАННЫЕ FERC/NERC, CEA, UCTE, Dulin et al. 2025 (Nature Comm.)
Скрипт: scripts/validate_real_events.py → results/validation_real_events.json
Совпадение секторов: 11/12 (92%)
Совпадение каскада: 4/4 (100%)
MAE = 0.203 Промах: Texas 2021, water — A[wat,ene]=0.006
WIOD не отражает физическую зависимость
насосов от электросети
6.2
Ключевой доказательный кейс: UCTE 2006
Малый шок энергетики (+0.05). Классический оператор с порогом $C_T = 0.928$ практически слеп. Количественный обнаруживает каскад в каждом прогоне. Реальность: каскад транспорта произошёл (подтверждено регулятором UCTE).
$K_{cl} = 0.038$ (классический слеп)
$K_q = 1.000$ (количественный обнаруживает)
$\Delta\% = +2532\%$ Механизм: $A[\text{tra},\text{ene}] = 0.500$
$\Delta x_T = 0.500 \times 0.667 \times 7\Delta t \approx 0.25 \gg \delta$
▼
Итог: $H_1$ подтверждена на 5/6 ненасыщенных сценариях ($\Delta\% = 45{-}2532\%$).
Валидация: 11/12 секторных пар (92%). Единственный промах — water←energy (WIOD-ограничение).
Область применимости: шоки умеренной интенсивности ($x_j + u_j < C_j$), при которых $A$ обеспечивает $\Delta x \geq \delta$.
