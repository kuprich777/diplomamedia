Дизайн эксперимента и методика проверки гипотезы $H_1$

УПРАВЛ. — варьируемый параметр
ФИКСИР. — контролируемый параметр
ВЫХОД — измеряемая величина
МЕТОД — инструмент
Фаза 0. Постановка задачи и фиксация критериев (до эксперимента)
0.1
Формулировка гипотезы
Фиксируются до получения данных: метрика, порог, условия. Исследовательская гипотеза: количественный оператор выявляет каскады чаще классического на ≥25%.
Критерии определяются заранее — это требование методологии (Новиков, 2010)
$$H_1: \frac{K^{(q)} - K^{(cl)}}{K^{(cl)}} \geq 0.25$$ $$H_0: \frac{K^{(q)} - K^{(cl)}}{K^{(cl)}} < 0.25$$
$K$ — полнота выявления каскадов (cascade detection rate)
0.2
Определение целевой функции сравнения
Метрика $K$ — доля MC-прогонов, в которых метод зафиксировал каскадный эффект, затрагивающий хотя бы одну неинициирующую подсистему. ВЫХОД
Метрика задана до эксперимента. Два индикатора: $I_{cl}$ (бинарный) и $I_q$ (количественный) — вычисляются параллельно на одних данных.
$$K^{(m)} = \frac{1}{N}\sum_{r=1}^{N} \mathbb{I}\{\text{каскад в прогоне } r\}$$ $I^{(q)} = \mathbb{I}\bigl(\exists\, i \neq i_0 : \max_t \Delta x_i \geq \delta\bigr)$ $I^{(cl)} = \mathbb{I}\bigl(\exists\, i \neq i_0 : \exists\, t : y_{i,t} = 1\bigr)$
$m \in \{cl, q\}$; $i_0$ — сектор-инициатор
0.3
Фиксация контролируемых параметров
Все параметры, которые не варьируются в рамках серии, фиксируются до запуска. ФИКСИР.
Фиксация обеспечивает воспроизводимость: повтор с тем же seed → тот же результат.
$\theta_{\text{node}} = 0.75$ — порог бинаризации $\delta = 0.10$ — порог каскада (колич.) $w = (0.4,\; 0.3,\; 0.3)$ — веса агрегирования $N = 1000$ — число MC-прогонов $\sigma_{\text{stoch}} = 0.30$ — масштаб шума
▼
Фаза 1. Калибровка матрицы зависимостей $A$
1.1
Prior: коэффициенты Леонтьева (WIOD 2016)
Извлекаются промежуточные потоки $X[i][j]$ из таблиц «затраты-выпуск» для трёх стран (РФ, Германия, США). Вычисляются коэффициенты прямых затрат, нормируются, усредняются. МЕТОД Анализ Леонтьева
Данные: WIOD 2016 NIOT (National Input-Output Tables), год 2014. Spearman $\rho = 0.962$ между странами.
$a_{\text{raw}}[i][j] = \dfrac{X[i][j]}{\text{GO}[j]}$ $a_{\text{norm}}[i][j] = \dfrac{a_{\text{raw}}[i][j]}{a_{\text{raw}}[j][j]}$ $\mu_0 = \frac{1}{3}(A_{\text{RUS}} + A_{\text{DEU}} + A_{\text{USA}})$ Rescale: $\max_{k \neq l} \to 0.5$
1.2
Likelihood: прокси-датасеты аварий
По первоисточникам (FERC/NERC, CEA) строятся почасовые ряды деградации $x_i(t)$ для Texas 2021 и India 2012. Из них оцениваются наблюдаемые коэффициенты связи. МЕТОД Прокси-реконструкция
$n = 214$ точек (168 Texas + 46 India). Шум $\sigma_\varepsilon = 1.2$.
$\bar{a}_{\text{data}}[i][j] \approx \dfrac{\sum_t \Delta x_i(t) \cdot x_j(t)}{\sum_t x_j(t)^2}$
Оценка из прокси-рядов (аналог OLS)
1.3
Posterior: байесовское обновление
Prior (WIOD) обновляется по likelihood (Texas + India) через сопряжённую нормальную модель. Каждый $a_{ij}$ получает постериорное распределение $\mathcal{N}(\mu_n, \sigma_n^2)$. МЕТОД Bayesian conjugate normal
$$\mu_n = \frac{\sigma_\varepsilon^2 \,\mu_0 + n\,\sigma_0^2\, \bar{a}_{\text{data}}}{\sigma_\varepsilon^2 + n\,\sigma_0^2}$$ $$\sigma_n^2 = \frac{\sigma_\varepsilon^2 \,\sigma_0^2}{\sigma_\varepsilon^2 + n\,\sigma_0^2}$$
$\mu_0, \sigma_0$ — prior; $n$ — число наблюдений; $\sigma_\varepsilon$ — шум данных
1.4
Проверка устойчивости: спектральный радиус
Вычисляются собственные значения $A_{\text{bayes}}$. Если $\rho(A) < 1$ — каскад затухает (аналог $\rho(W) < 1$ в DebtRank). ВЫХОД
$\rho(A_{\text{bayes}}) = 0.194$ → каскад затухает за 3–4 шага. Модель корректна.
$\det(A - \lambda I) = 0$
$\lambda^3 - 0.0368\lambda - 0.0002 = 0$ $\rho(A) = \max|\lambda_i| = 0.194 < 1$ ✓
Перрон–Фробениус: $\rho < 1 \Rightarrow$ сходимость итеративного оператора
▼
Фаза 2. Один прогон Monte Carlo (повторяется $N = 1000$ раз)
2.1
Генерация seed
Для каждого прогона $r$ вычисляется детерминированный seed. Это гарантирует воспроизводимость: одинаковые $(s, r)$ → одинаковый результат. ФИКСИР.
$$\text{seed}(s, r) = \text{int}\!\left(\text{SHA256}(s : r)_{[0:16]},\; 16\right)$$
$s$ — scenario_id, $r$ — run_id
2.2
Инициализация начального состояния $x_0$
Доменные микросервисы возвращают базовое состояние каждого сектора. Это состояние «до аварии». ФИКСИР.
Рассчитывается по формулам доменных моделей (consumption/production для energy, demand/supply для water и т.д.)
$x_0 = (x_{\text{ene}},\; x_{\text{wat}},\; x_{\text{tra}})$ Пример: $x_0 = (0.667,\; 0.055,\; 0.333)$ $R_0 = \text{Agg}(x_0, w) = \sum_i w_i \cdot x_{i,0}$
2.3
Задание нагрузки (шока) $u_t$
Из каталога сценариев берётся базовый load_amount. К нему добавляется гауссовский шум, моделирующий неопределённость реальной аварии. УПРАВЛ. load_amount УПРАВЛ. stochastic_scale
Откуда load: задаётся исследователем при запуске MC. Значения калиброваны по первоисточникам (Baltimore 0.30, India 0.58, Texas 0.70) или перебираются (severity sweep 0.10–0.80).
$u_t^{\text{base}}$ ← load_amount из запроса MC $\varepsilon \sim \mathcal{N}(0,\; (\sigma_{\text{stoch}} \times u^{\text{base}})^2)$ $$u_t^{\text{actual}} = \max(0,\;\; u_t^{\text{base}} + \varepsilon)$$
Пример: load=0.40, σ=0.30 → $\varepsilon \sim \mathcal{N}(0, 0.12^2)$ → $u \in [0.28, 0.52]$ в 68% прогонов
2.4
Применение шока к доменному сервису
Стенд отправляет HTTP-запрос целевому микросервису. Сервис пересчитывает внутреннее состояние по доменной формуле.
Каждый сектор имеет свою формулу: energy — (consumption/production − 0.7)/0.3; transport — $1 - e^{-3 \cdot \text{load}}$; water — (demand − supply)/demand.
POST /api/v1/transport/increase_load?amount=$u_t$ $x_{\text{tra}} = 1 - e^{-3 \cdot u_t}$ Пример: $u_t = 0.40 \Rightarrow x_{\text{tra}} = 1 - e^{-1.2} = 0.699$
2.5
Каскадное распространение (BFS-очередь)
Risk engine запрашивает обновлённое состояние всех секторов, затем применяет матрицу $A$ для распространения шока на смежные отрасли. ФИКСИР. Матрица $A$
$$\mathbf{x}_{t+1} = \text{clip}_{[0,1]}\!\left(\mathbf{x}_t + \mathbf{u}_t + A\mathbf{x}_t\right)$$ Покомпонентно:
$x_{i,t+1} = \text{clip}\!\left(x_{i,t} + u_{i,t} + \sum_j a_{ji} \cdot x_{j,t}\right)$
$t$ — дискретный шаг итерации каскадного процесса
2.6
Вычисление индикаторов каскада
Параллельно для одних и тех же данных вычисляются два индикатора: классический (бинарный) и количественный (непрерывный). ВЫХОД $I_{cl}$, $I_q$ для данного прогона
Количественный: $\Delta x_i = x_{i,\text{after}} - x_{i,\text{before}}$
$I_q = \begin{cases} 1, & \text{если } \exists\, i \neq i_0 : |\Delta x_i| \geq \delta \\ 0, & \text{иначе} \end{cases}$ Классический: $y_i = \mathbb{I}(x_i \geq \theta_{\text{node}})$
$I_{cl} = \begin{cases} 1, & \text{если } \exists\, i \neq i_0 : y_{i,\text{after}} = 1 \\ 0, & \text{иначе} \end{cases}$
2.7
Фиксация результатов прогона
Сохраняются: вектора до/после, индикаторы, прирост интегрального риска, траектория. ВЫХОД
Каждый прогон — одна строка в JSON-массиве results.
Запись прогона $r$:
$(I_{cl,r},\;\; I_{q,r},\;\; \Delta R_r,\;\; x_{\text{before}},\;\; x_{\text{after}})$ $\Delta R_r = R_{\text{after}} - R_{\text{before}} = \text{Agg}(x_{\text{after}}, w) - \text{Agg}(x_{\text{before}}, w)$
▼ повторить $N = 1000$ раз
Фаза 3. Агрегация и измерение метрик
3.1
Агрегация полноты выявления $K$
По $N$ прогонам считается доля тех, где каскад был обнаружен — отдельно для каждого метода. ВЫХОД $K_{cl}$, $K_q$
$$K_{cl} = \frac{1}{N}\sum_{r=1}^{N} I_{cl,r}$$ $$K_q = \frac{1}{N}\sum_{r=1}^{N} I_{q,r}$$
3.2
Вычисление статистик прироста риска
Распределение $\Delta R$ характеризует «хвосты» — редкие высокорисковые реализации. ВЫХОД
$\overline{\Delta R} = \frac{1}{N}\sum_{r=1}^{N} \Delta R_r$ $p_{95}(\Delta R)$ — 95-й перцентиль
▼
Фаза 4. Проверка гипотезы — решение о $H_0$
4.1
Вычисление критерия
Относительный прирост $\Delta\%$ сравнивается с порогом 25%, зафиксированным до эксперимента.
$$\Delta\% = \frac{K_q - K_{cl}}{K_{cl}} \times 100\%$$
4.2
Решение
Если $\Delta\% \geq 25\%$ — $H_0$ отклоняется в пользу $H_1$. Если $\Delta\% < 25\%$ — $H_0$ не отклоняется.
Решение принимается для каждого сценария отдельно — это не единый статистический тест, а серия парных сравнений при контролируемых условиях.
$\Delta\% \geq 25\%$ → $H_0$ ОТКЛОНЯЕТСЯ ✓
(количественный метод превосходит классический) $\Delta\% < 25\%$ → $H_0$ не отклоняется
(превосходство не достигает порога)
4.3
Пример: подстановка результатов
Конкретные числа для каждого сценария.
S4: $\frac{0.798 - 0.353}{0.353} = +126\%$ ≥ 25% → ✓ Baltimore: $\frac{0.880 - 0.090}{0.090} = +878\%$ ≥ 25% → ✓ S3: $\frac{0.368 - 0.331}{0.331} = +11\%$ < 25% → ✗
▼
Фаза 5. Анализ чувствительности (робастность результатов)
5.1
Severity sweep
Повторение фаз 2–4 при $\text{load} \in \{0.10, 0.15, ..., 0.60\}$. Цель — определить зону максимального разрыва $\Delta K$. УПРАВЛ. load_amount
$N = 500$ прогонов на точку × 8 точек = 4000 прогонов.
$K_q(\text{load}),\;\; K_{cl}(\text{load})$
$\Delta K(\text{load}) = K_q - K_{cl}$
Результат: макс. $\Delta K \approx 0.47$ при load $\in [0.15, 0.25]$
5.2
$\theta$-sweep
Повторение при $\theta \in \{0.50, 0.55, ..., 0.90\}$. Цель — показать, что $K_q$ инвариантен к $\theta$, а $K_{cl}$ критически зависит. УПРАВЛ. θ_node
$\frac{\partial K_q}{\partial \theta} \approx 0$ (инвариантность) $\frac{\partial K_{cl}}{\partial \theta} \neq 0$ (критическая зависимость)
$\theta^* = 0.75$ — оптимальный порог для обоих сценариев
5.3
Валидация на исторических кейсах
Прогон модели на прокси-датасетах реальных аварий (Baltimore, India, Texas, Europe). Цель — подтвердить режимную согласованность. УПРАВЛ. scenario МЕТОД Историко-аналоговая валидация
Baltimore: $K_{cl}=0.09$, $K_q=0.88$ → $\Delta\%=+878\%$ ✓ India: $K_{cl}=0.35$, $K_q=1.00$ → $\Delta\%=+187\%$ ✓ Texas: $K_{cl}=K_q=1.00$ → насыщение Europe: $K_{cl}>K_q$ → структурное ограничение
▼
Итог: $H_1$ подтверждена для S4, Baltimore, India ($\Delta\% \gg 25\%$). S3 — пограничный. Europe — структурное ограничение.
